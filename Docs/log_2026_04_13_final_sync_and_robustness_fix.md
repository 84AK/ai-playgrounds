# 📝 개발 로그 (2026-04-13) - 커리큘럼 연동 및 성능 최적화

## 1. 개요
커리큘럼 시트 데이터 연동 실패 및 학습 대시보드(Study Lab)의 무한 로딩 문제를 해결하고, 전체적인 시스템 안정성과 보안을 검토했습니다.

## 2. 주요 수정 내역

### 🛠 시트 연동 오류 (Critical Fix)
- **증상:** 구글 시트(`CourseContents`)에 데이터가 있음에도 화면에는 로컬 `.md` 파일만 출력됨.
- **원인:** 백엔드(GAS) 코드 내에서 존재하지 않는 함수 `s.getValues()`를 호출하여 `TypeError` 발생. (데이터를 가져오지 못해 프론트엔드에서 로컬로 폴백됨)
- **해결:** `s.getDataRange().getValues()`로 명령어를 수정하여 시트 전체 데이터를 정확히 로드하도록 개선.
- **추가 개선:** 주차(Week) 데이터 매칭 시 숫자형과 문자열형이 혼용되어도 인식할 수 있도록 `parseInt()` 및 `toString().trim()` 처리 보강.

### ⚡ 학습 대시보드 무한 로딩 최적화
- **증상:** `My Study Lab` 진입 시 프로그레스 바 로딩이 멈추지 않고 터미널에 무한한 API 요청 로그 발생.
- **원인:** `useSWR`의 fetcher 함수가 렌더링 시마다 새로 생성되어 의존성 루프 발생.
- **해결:** `useCallback`을 사용하여 fetcher 함수를 메모리화하여 불필요한 재호출 방지.

### 🛡 보안 및 유효성 검사
- `.env.local` 파일이 `.gitignore`에 정상 등록되어 환경 변수 유출 방지 확인.
- `APPS_SCRIPT_URL` 호출 시 서버 사이드 프록시(/api/proxy-apps-script)를 거치도록 하여 브라우저 노출 최소화.
- 관리자용 디버그 도구(Admin Debug Badge)를 통해 연동 상태를 실시간 진단하고, 문제 해결 후 코드를 제거하여 보안성 유지.

## 3. 회고 및 권장 사항
- **GAS 마스터 코드 배포:** 향후 시트 구조가 바뀌거나 연동 문제가 발생하면 설정 페이지(/admin/setup)에서 제공하는 최신 마스터 코드를 다시 배포하는 것만으로 대부분의 로직 문제를 해결할 수 있습니다.
- **디버깅 팁:** 데이터가 안 나올 때는 화면 왼쪽 아래 디버그 모드(현재는 제거됨)나 터미널의 `[CourseContent] Response Data` 로그를 가장 먼저 확인하세요.
