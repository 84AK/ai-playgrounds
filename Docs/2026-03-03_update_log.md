# 2026-03-03 MBTI Maker UI 및 버그 긴급 수정 기록 (Update Log)

## 📌 작업 개요
* **담당자:** 서기 (Scribe)
* **작업 대상:** `app/mbti/page.tsx`, `app/mbti/result/page.tsx`
* **주요 목적:** 사용자 피드백을 반영하여 오해를 일으키는 UI 요소를 제거하고, 치명적인 프리뷰 에러를 방지함.

## 🛠 수정 & 에러 해결 내역

### 1. 프리뷰 화면 기본 텍스트 수정 (UI Polish)
* **문제점:** 결과 데이터 없이 프리뷰 실행 시 "당신은 결과 이름입니다"라는 어색한 기본값이 노출됨.
* **해결 방안:** 작성되지 않은 결과값에 대해 "아직 직함이 없습니다", "작성해주신 결과 설명이 이 곳에 표시됩니다"로 대체 텍스트를 수정하여 제작자가 명확히 인지할 수 있도록 개선함.

### 2. 눈속임용 UI 제거 (UI Polish)
* **문제점:** 미완성 기능인 'SAVE DRAFT' 버튼과 'AI LABORATORY REAL-TIME CLOUD SYCING...' 장식용 메세지가 혼란을 줌.
* **해결 방안:** 두 UI 요소를 `app/mbti/page.tsx` 하단 바에서 완전히 제거하고 '쇼케이스에 저장하기' 버튼을 우측 정렬함.

### 3. 데이터 로딩 시 화이트 스크린 크래시 에러 방어 (Fix / Solver)
* **문제점:** 저장된 데이터(questions, results)가 구조에 맞지 않거나 비어있는 빈 배열일 경우, 프리뷰 화면을 렌더링하다가 `questions[previewIndex]?.text` 등을 참조하지 못해 전체 페이지가 하얗게 다운(Crash)됨.
* **해결 방안:** 데이터 파싱 블록(`typeof userProject.questions === 'string'`) 하단에 `Array.isArray(parsedQuestions) && parsedQuestions.length > 0` 조건문을 추가하여, 유효한 데이터가 있을 때만 `setQuestions`, `setResults` 상태를 덮어쓰도록 안전망(Safeguard)을 구축함.

### 4. 쇼케이스 저장 라우팅 오류 수정 (Fix / Solver)
* **문제점:** '쇼케이스에 저장하기' 클릭 시 `/mbti/result?type=...` 결과 페이지로 강제 이동됨.
* **해결 방안:** `window.location.href = '/showcase'` 로 코드를 수정하여 정상적으로 전시관 페이지로 넘어가도록 처리함.

### 5. 결과 화면 (Result) UI 및 문구 개선 (UI / UX Polish)
* **문제점:** 일반 사용자가 보는 `app/mbti/result/page.tsx` 화면에서 '우리 프로젝트 전시관에 공유하기 ✨' 및 임시 저장용 얼럿창(alert) 띄우는 더미 버튼(💾)이 어색함.
* **해결 방안:** 저장 버튼은 제거하고, "우리 프로젝트 전시관에 공유하기 ✨" 문구를 "다른 작품들 구경가기 ✨"로 변경하여 자연스러운 플로우를 유도함.

### 6. 프리뷰 내부 실제 결과 계산 로직 추가 (Relector / Blueprint)
* **문제점:** 프리뷰 모드의 문항 응답에 상관없이 고정된 이름순 결과만 출력됨.
* **해결 방안:** `previewScores` 상태 값을 추가하고 보기 버튼을 누를 때마다 각 축(E/I, S/N, T/F, J/P)의 점수를 집계하여, 최종적으로 사용자가 누른 값을 토대로 산출된 `previewResultType`을 노출하도록 개선함. 또한 "입니다입니다!" 등 중복되는 종결어미를 정돈함.

### 7. Google Sheets 저장 로직 및 안전 안내 UI (Relector / UI Polish)
* **문제점:** 작성자가 쇼케이스에 저장하기 전 페이지를 이탈하면 데이터가 유실됨. 또한 명시적 안내가 없었음.
* **해결 방안:**
  - 글로벌 액션 바(하단 UI) 상단에 붉은색 경고 박스(`데이터가 아직 완전히 저장되지 않았습니다!`)를 배치함. 
  - 버튼을 누를 때 `isSaving`, `isSaved` 상태를 관리하여 로딩 스피너를 보여주고, 저장이 끝나면 "저장 완료! 전시관 구경하기 ✨"로 안내 텍스트가 변경되며 자동으로 전시관에 넘어가도록 수정됨.
  
### 8. My Study Lab 진척도 관리 및 Google Drive 파일 업로드 연동 강화 (Blueprint / Worker)
* **문제점:** 홈페이지에서 단순 클릭만으로 100% 진도를 무분별하게 달성할 수 있었음.
* **해결 방안:**
  - 메인 홈페이지의 항목을 클릭하면 각 주차별 전용 학습 페이지(`app/course/[weekId]`)로 진입하는 커리큘럼 기반 라우팅 시스템 구현.
  - 마크다운 파싱 플러그인(`react-markdown`, `@tailwindcss/typography`)을 도입하여 노션처럼 깔끔한 수강 화면 렌더링.
  - 코스 하단에 **구글 드라이브 기반의 파일 업로드 기능** 추가! 직접 짠 코드 파일(htmk, css 등)을 제출하여 Base64로 Apps Script에 전송, 사용자 구글 드라이브 지정 폴더에 저장되게 연동.
  - 파일 제출이 완료되어야만 My Study Lab의 달성 퍼센트가 올라가는 교육적인 플로우 완성.
