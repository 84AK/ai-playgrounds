# 2026-03-05 작업 로그

## 🛠 수정 및 에러 해결 내용 (Counselor & Fix & UI Polish & Doc 협업)

### 1. 비밀번호 입력 가시성 개선 (UX)
- **문제 발생**: 실전 수업 중 특정 인원만 회원가입/로그인 시 "비밀번호가 일치하지 않는다"는 에러를 겪음. 주요 원인은 스마트폰 환경에서의 자동 띄어쓰기(공백 추가), 첫 글자 대문자 자동 변환, 혹은 오타로 판단됨.
- **해결 방안**: 사용자가 자신이 입력한 암호를 눈으로 직접 확인할 수 있도록 비밀번호 보기 토글 버튼(👁️) 추가 구현.
- **관련 파일**:
  - `components/GlobalAuthGuard.tsx`: 로그인/가입 폼에 `showPassword` 상태 및 토글 버튼 추가.
  - `components/UserSettingsModal.tsx`: 내 정보 잠금 해제용 `showPasswordUnlock` 및 비밀번호 변경용 `showPasswordEdit` 상태 추가 및 토글 버튼 컴포넌트 적용.

### 2. 에러 메시지 친화적 보강
- **문제 발생**: 기존의 에러 메시지는 너무 단순하여 문제의 명확한 원인 파악과 자가 해결을 돕지 못함.
- **해결 방안**: 팝업창 모달의 에러 표시 텍스트에, 띄어쓰기 및 대문자 입력 관련 체크리스트를 포함하여 안내함.
- **수정 내용**:
  - 기존: *"비밀번호가 일치하지 않습니다."*
  - 변경: *"비밀번호가 일치하지 않습니다.\n(끝에 띄어쓰기가 포함되었거나, 첫 글자가 대문자로 입력되지 않았는지 확인해 주세요!)"*

### 3. 쇼케이스 MBTI 아이템 설명 누락 버그 픽스
- **문제 발생**: 쇼케이스 등록 시 커스텀 앱들은 세부 설명을 가지지만, MBTI 결과 목록에서 연동되는 앱들은 별도의 상세 설명이 없어 모두 "이 작품에 대한 상세 설명이 아직 등록되지 않았습니다."(혹은 커스텀 앱 등록 시 다른 작성자의 설명)로 동일하게 노출되는 현상 발견.
- **해결 방안**: `app/showcase/page.tsx`에서 MBTI 데이터를 가져와 쇼케이스 고유 맵에 매핑할 때, `description: \`\${type} 유형의 성격 테스트입니다.\`` 값을 기본으로 할당해주도록 로직 수정.

### 4. MBTI 결과 화면 초기 기본값 노출 문제 (Flickering UX 개선)
- **문제 발생**: `app/mbti/result/page.tsx` 진입 시, 커스텀 결과표 데이터를 비동기로 불러오는(`isFetching`) 과정 동안 `type`값에 해당하는 기본 텍스트(예: "낭만적인 중재자")가 0.5초가량 먼저 노출되었다가 커스텀 텍스트(예: "급식실의 낭만가")로 순식간에 변경되는 번쩍임(Flickering) 현상 발생.
- **해결 방안**: `isFetching` 상태 변수를 추가하고, `fetchCustomResults` 함수가 완전히 동작을 마치기 전까지는 "Analyzing DNA..." 등의 로딩 안내 메시지를 띄우는 방향으로 제어 로직을 보강함.

### 5. 쇼케이스 카테고리 표시 및 MBTI 필터링 누락 버그 해결
- **문제 발생**: 새 작품을 전시할 때 카테고리(MBTI 등)를 명확히 선택했음에도 불구하고, 쇼케이스 메인 페이지 상단의 'MBTI' 필터링 탭을 눌러도 해당 작품이 뜨지 않았고, 카드 내의 태그 또한 노출되지 않는 문제 발생. 구글 앱스 스크립트에서 응답받은 배열을 프론트엔드가 파싱하여 무조건 `type: "CUSTOM"`으로 강제 덮어쓰기 하고 있었음.
- **해결 방안**: `app/showcase/page.tsx`에서 `type: item.Type || item.type || "CUSTOM"` 와 같이 원본 데이터의 타입을 우선하도록 고치고, 카드 내의 숨김 처리된 뱃지 태그를 복원하여 `type` 값에 따라 각기 다른 디자인(MBTI는 보라색 뱃지 등)이 나오도록 스타일링 보강.

## 📝 다음 작업 참고용 기록
- 사용자 입력 단계에서 발생하는 오타 빈도를 줄일 수 있는 확실한 초기 대처법입니다.
- 백엔드나 전역 설정 등에서 사전에 입력값의 공백을 제거(`trim()`)하는 로직은 이미 일부 적용되어 있으나, 첫 글자 대문자 문제 등은 사용자가 직접 인지하는 것이 가장 좋으므로 현재의 UI/UX 개선 방향이 적합합니다.
- 데이터 로딩 시 스켈레톤이나 로딩 모달을 사용하여 기본값이 노출되어 몰입감을 깨는 UX를 지속적으로 방지해야 합니다.
