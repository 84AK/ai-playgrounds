  - 푸터의 `Curriculum` 링크를 누르면 `docs/mbti_week_plan.md`와 `docs/pose_week_plan.md` 문서를 결합해 전체 커리큘럼을 보여주는 `/curriculum` 페이지 개설.
  - 푸터의 `Guide` 링크를 누르면 플랫폼 전반의 사용 안내서 역할을 하는 `docs/guide.md` 를 보여주는 `/guide` 페이지 신설 개통(Markdown 뷰어 컴포넌트 활용).
- **쇼케이스(Showcase) 외부 링크 최적화:** '프로젝트 직접 체험하러 가기' 링크 버튼 클릭 시 항상 새 탭으로 열렸던 속성(`target="_blank"`)을 제거하여 한 창(Same tab)에서 서비스가 연결되도록 UX 이음새 강화.
- **사용자 캐릭터(Avatar) 옵션 확장:** 연구소 프로필 설정(회원가입, 정보수정 모달) 화면에서 선택할 수 있는 귀여운 동물 이모지 컬렉션을 기존 6개에서 12종류(`🐶`, `🐱`, `🐰`, `🦊`, `🐻`, `🐼`, `🦁`, `🐯`, `🐨`, `🐸`, `🐹`, `🐵`)로 확대, 모달창 4열/6열 정렬 디자인 호환 수정.

## 🐛 버그 및 에러 해결 (Fix & Solves)
- **[Fix]** MBTI Maker "데이터 불러오기" 클릭 시 데이터 형식 불일치로 발생하던 오류해결 (`reverse is not a function`). Google Sheets 응답 데이터의 키(대소문자 구분 및 형태) 변화에 유연하게 대응하도록 파싱 분기 처리 로직 안정화.
- **[Fix]** Play 화면에서 Google Sheets의 MBTI 정보를 불러올 때 "Project not found" 에러창이 뜨던 문제 해결 (`searchParams`와 API 데이터 간의 저자명 `Author` 필드값 파싱 매핑 로직 수정).
- **[Fix]** 쇼케이스 상세 보기 팝업에서 이모지로 만들어진 썸네일 이미지가 깨져 보이던(`broken img tag`) 현상을 고치고, 이모지일 때는 HTML Text 형태로 크고 선명하게 출력하도록 예외 처리.
- **[Optimize]** "테스트 다시하기" 버튼 클릭 시 구글 시트에서 API 데이터를 처음부터 다시 불러와 약 15초가량 딜레이되던 비효율을, 브라우저 `localStorage` 기반 상태 캐싱(Caching)을 통해 **버튼 클릭 즉시 테스트 화면이 리셋(0초 로딩)** 되도록 획기적으로 개선.
