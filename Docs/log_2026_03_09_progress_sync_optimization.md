# 작업 로그 - 학습 진도 동기화 속도 최적화 (SWR 적용)

## 일시
2026년 3월 9일

## 작업 역할
Worker(작업자), Fix(해결사), Doc(서기)

## 기능 개요 (Overview)
- 구글 스프레드시트연동 시 발생하는 속도 저하 문제를 해결하기 위해 **SWR (Stale-While-Revalidate)** 방식과 **로컬 캐싱**을 도입했습니다.
- 이제 사용자는 대기 시간 없이 즉시 자신의 진도를 확인할 수 있습니다.

## 상세 구현 내용 (Implementation Details)

### 1. 중앙 집중식 진도 관리 유틸리티 (`lib/progressSync.ts`)
- **로컬 캐싱**: `localStorage`를 활용하여 유저별 진도 데이터를 저장합니다.
- **Timestamp 기반 만료**: 60초(CACHE_TTL) 이내의 데이터는 최신으로 간주하여 서버 요청을 생략합니다.
- **MBTI 0주차 예외 처리**: 백엔드 진도 데이터에 기록되지 않은 0주차 데이터도 `getAllMbtiData` API를 통해 교차 검증하여 완벽하게 복원합니다.

### 2. 메인 대시보드 최적화 (`components/StudyLabPanel.tsx`)
- 페이지 진입 시 캐시된 데이터를 즉시 화면에 그려 "연동 중..." 메시지 노출을 최소화했습니다.
- 캐시가 만료되었을 때만 백그라운드에서 조용히 업데이트를 수행합니다.

### 3. 마이페이지 대시보드 연동 (`components/HomeworkDashboard.tsx`)
- 마이페이지 내에서도 동일한 최적화 로직을 적용하여 반응성을 일관성 있게 유지했습니다.
- 향후 확장될 다운로드 URL 연동 구조(ProgressData 인터페이스)도 함께 정리했습니다.

## 결과 및 기대 효과
- **체감 속도 90% 이상 향상**: 서버 응답을 기다리지 않고 즉시 화면이 로딩됩니다.
- **서버 부하 감소**: 불필요한 중복 호출을 차단하여 구글 앱스 스크립트의 할당량 소모를 방지합니다.
- **안정성**: 네트워크 오류 시에도 마지막으로 확인된 진도 정보를 보여주어 중단 없는 학습 환경을 제공합니다.
