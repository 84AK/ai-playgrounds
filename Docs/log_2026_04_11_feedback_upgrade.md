# 작업 로그: 주차별 피드백 시스템 고도화 (v6.0)

## 1. 개요
기존에 전역(Global) 컬럼 하나로 관리되던 피드백 시스템을 주차별/코스별로 분리하여 저장하고 조회할 수 있도록 고도화함.

## 2. 발생한 문제 및 해결 방안
- **문제 1**: 특정 주차에 피드백이 없을 때 과거 다른 주차의 피드백이 보여지는 현상.
    - **해결**: `checkUserStatus`에서 특정 주차 컬럼만 조회하도록 수정하고, 전용 피드백 컬럼이 비어있을 때 전역 컬럼을 참조하던 Fallback 로직을 제거함.
- **문제 2**: `Users` 시트에 피드백이 단일 컬럼으로 되어 있어 주차별 기록이 불가능함.
    - **해결**: `Users` 시트 헤더를 확장하여 MBTI(4주), POSE(4주) 총 8개의 피드백 전용 컬럼을 생성함.

## 3. 수정 및 구현 내역

### 백엔드 (Apps Script v6.0)
- **시트 헤더 확장**: `User_ID`부터 `POSE_W4`까지 총 15개 컬럼으로 구성.
- **`updateFeedback`**: `week`와 `course_type` 파라미터를 기반으로 적절한 열 번호(Col 8~15)를 계산하여 기록.
- **`checkUserStatus`**: 요청된 주차에 매핑되는 컬럼만 읽어서 반환.
- **`getStudentList`**: 한 번의 호출로 모든 주차의 피드백 상태를 알 수 있도록 `feedbacks` 객체 구조를 추가하여 반환.

### 프론트엔드 (Next.js)
- **`app/admin/feedback/page.tsx`**: 
    - `Student` 인터페이스에 `feedbacks` 속성 추가.
    - 학생 목록 렌더링 시, 현재 관리자가 선택한 **주차**와 **코스**에 피드백이 있는지 실시간으로 체크하여 "피드백 있음" 배지를 표시하도록 로직 개선.
- **`UploadHomework.tsx`**: 백엔드 로직 변경에 따라 자연스럽게 해당 주차의 피드백만 표시됨을 확인.

## 4. 시트 컬럼 매핑 가이드 (1-indexed)
- 8번: MBTI 1주차
- 9번: MBTI 2주차
- 10번: MBTI 3주차
- 11번: MBTI 4주차
- 12번: POSE 1주차
- 13번: POSE 2주차
- 14번: POSE 3주차
- 15번: POSE 4주차

## 5. 결과 및 기대 효과
- 학생들은 각 주차에 맞는 정확한 피드백만 확인할 수 있어 혼란이 제거됨.
- 관리자는 시트 하나에서 모든 주차의 피드백 진행 상황을 체계적으로 관리할 수 있음.
