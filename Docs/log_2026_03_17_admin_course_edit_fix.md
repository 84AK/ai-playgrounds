# 2026-03-17 Admin Course Edit Bug Fix Log

## 1. 개요 (Overview)
- **발생한 문제**: 
  1. 관리자 주차별 코스 수정 페이지에서 다른 주차(예: 1주차 -> 2주차)를 클릭해도 에디터 입력창의 내용이 이전 주차 내용으로 고정되어 바뀌지 않는 문제.
  2. 수정한 내용을 저장하려고 할 때 "파일 저장 중 오류가 발생했습니다." 라는 메시지와 함께 저장이 실패하는 문제.
- **작업자**: 해결자(Fix), 서기(Doc)

## 2. 에러 분석 (Error Analysis)
- **상태 업데이트 문제**: `AdminCourseEditorPanel.tsx` 컴포넌트가 처음 렌더링될 때 `initialContent`를 상태(state)로 설정하지만, 이후 주차(`weekId`)가 변경되어도 컴포넌트가 언마운트되지 않아 상태가 최신 `initialContent`로 갱신되지 않는 리액트의 전형적인 상태 비동기화 문제.
- **저장 실패 문제**: `app/api/course/save/route.ts` API에서 `try-catch`로 에러를 잡았을 때 일괄적으로 "파일 저장 중 오류가 발생했습니다." 메시지를 띄우고 있었음. 개발 환경이나 파일시스템 권한 등 다양한 원인이 있을 수 있으나 구체적인 에러 메시지를 은닉하고 있어 원인 파악이 어려웠음.

## 3. 해결 및 구현 내용 (Resolution & Implementation)
- **UI 상태 갱신 Fix**: 
  - `app/admin/course/page.tsx`에 있는 `<AdminCourseEditorPanel>` 렌더 부분에 `key={`${track}-${weekId}`}` 속성을 추가.
  - 이를 통해 트랙이나 주차가 바뀔 때마다 컴포넌트가 새로 마운트되어 초기값이 올바르게 할당되도록 수정.
- **저장 에러 메세지 구체화**:
  - `route.ts`의 `catch` 블록에서 반환하는 에러 메시지에 실제 에러 내용(`error.message`)을 덧붙이도록 수정하여, 향후 저장에 다시 실패하더라도 사용자 화면에 정확한 원인(예: ENOENT, EACCES 등)이 출력되도록 개선.

## 4. 향후 작업 및 참고 사항
- 이번 수정으로 주차를 변경할 때마다 해당 주차의 내용이 에디터에 정상적으로 표시될 것입니다.
- 만약 저장 시 다시 오류 메시지가 발생한다면, 이번에 추가한 구체적인 에러 내용(예: "파일 저장 중 오류가 발생했습니다: ...")을 확인하여 후속 조치(폴더 권한 수정 또는 Apps Script 연동 문제 확인 등)를 진행해야 합니다.
