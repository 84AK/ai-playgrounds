# 작업 로그: 무한 확장형 동적 교육과정 및 피드백 시스템 (v7.0)

## 1. 개요
관리자가 주차를 추가하면 시스템이 자동으로 감지하여 시트를 넓히고 UI를 업데이트하는 **'무한 확장형(Dynamic Scaling)'** 구조를 도입함.

## 2. 주요 개선 사항

### 백엔드 (Apps Script v7.0)
- **안정적 버퍼 (20주차)**: `Progress` 시트의 URL 저장 위치가 꼬이지 않도록 20주차 분량의 컬럼을 미리 확보하고, URL 오프셋을 고정함.
- **주차별 피드백 분리**: `MBTI_W1`~`POSE_W20`까지 총 40개의 피드백 전용 컬럼 공간을 확보하여 데이터 간섭을 완벽히 차단함.
- **동적 컬럼 매핑**: 요청받은 주차 번호에 따라 런타임에 컬럼 위치를 계산하여 저장 및 조회함.

### 프론트엔드 (Admin Dashboard)
- **커리큘럼 연동**: 피드백 선택기의 주차 목록을 `getAllCourseStructure` API와 연동함. 
    - 이제 관리자가 13주차 수업 내용을 등록하면, 피드백 페이지에서도 즉시 13주차 선택 버튼이 나타남.
- **실시간 피드백 감지**: `feedbacks` 객체 구조를 통해 현재 선택된 학생이 특정 주차/코스에 피드백을 받았는지 즉시 파악하여 UI에 반영함.

## 3. 시트 컬럼 상세 설계 (v7.0)

### [Users] 시트 (피드백 전용)
- 1~7: 기본 정보 (ID, 학교, 비밀번호, 아바타, 수정일, 학년, 반)
- 8 ~ 27: MBTI 1~20주차 피드백
- 28 ~ 47: POSE 1~20주차 피드백

### [Progress] 시트 (진도 및 과제 전용)
- 1: User_ID
- 2 ~ 21: 1~20주차 과제 제출 여부 (Status)
- 22 ~ 41: 1~20주차 과제 파일 URL

## 4. 기대 효과
- **운영 자동화**: 새로운 주차가 추가되어도 개발자의 개입 없이 시스템이 알아서 확장됨.
- **데이터 안정성**: 주차가 늘어남에 따라 기존 데이터(URL, 피드백)가 밀리는 현상을 원천 방지함.
