# 프로젝트 작업 로그 (2026-03-17)

## 📋 이슈 사항 (Issue)
- **에러 메시지**: `EROFS: read-only file system, open '/var/task/Docs/mbti_week2.md'`
- **상황**: 관리자 페이지에서 2주차 강의 콘텐츠를 저장할 때 에러 발생 및 구글 스프레드시트 저장 확인 불가.
- **원인 분석**:
  1. 기존 `lib/courseContent.ts` 코드는 구글 스프레드시트(Apps Script) 연동 실패 시 로컬 파일(`Docs/*.md`)에 백업 및 로드를 시도하는 **Fallback(대체) 로직**이 존재했음.
  2. Vercel 서버리스 환경은 파일 쓰기가 금지(`Read-Only`)되어 있어, 로컬 저장 시도 시 `EROFS` 에러를 뿜어냄.
  3. 또한 시트에 내용이 비어있을 때 로컬 템플릿 파일을 읽어와 보여주는 바람에 "입력하지 않은 내용"이 편집판에 노출되는 부작용이 있었음.

## 🛠️ 해결 및 구현 (Resolution)
- **수정 파일**: `lib/courseContent.ts`
- **적용 내용**:
  1. **Source of Truth 강화**: `APPS_SCRIPT_URL`이 있을 경우 오직 구글 스프레드시트만 사용하도록 강제.
  2. **가져오기 (`getCourseContent`)**: 구글 시트에 내용이 없으면 로컬 템플릿을 읽지 않고 **빈 문자열(`""`)**을 반환하도록 수정하여 혼란 방지.
  3. **저장하기 (`saveCourseContent`)**: 스프레드시트 저장 실패 시 로컬 파일 쓰기로 우회하지 않고, **명확한 에러(`throw new Error`)**를 발생시켜 트래킹이 가능하도록 구조 변경.
  4. Vercel 환경에서 안전하게 동작하며 서버 에러 로그 및 클라이언트 토스트 메시지 등으로 정확히 어떤 오류(인증 실패, 시트 누락 등) 피드백을 받을 수 있게 됨.

## 🔍 회고 및 참고 (Retrospectives)
- 로컬 개발 편의성을 위한 Fallback 로직이 프로덕션(Vercel) 환경에서 예기치 못한 에러와 데이터 불일치 착각을 불러일으킬 수 있음을 확인.
- 클라우드 서버리스 환경 배포 시 `fs` 모듈 사용에 항상 주의가 필요함.
