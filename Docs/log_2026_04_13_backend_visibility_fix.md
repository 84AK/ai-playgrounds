# 작업 로그: 백엔드 출처 가시화 및 초기화 기능 강화 (2026-04-13)

## 1. 개요
선생님들이 매직 링크나 개인 설정을 통해 개별 구글 시트를 연동했을 때, 현재 어떤 환경을 사용 중인지 명확히 알 수 있게 하고, 문제 발생 시 즉시 시스템 기본값으로 복구할 수 있는 장치를 마련했습니다.

## 2. 주요 변경 사항

### A. 백엔드 엔진 고도화 (`lib/gasTemplate.ts`)
- `testConnection` 액션 호출 시 단순히 성공 메시지만 주는 것이 아니라, 현재 연결된 스프레드시트의 이름(`ss.getName()`)을 함께 반환하도록 수정했습니다.

### B. 관리자 설정 페이지 개선 (`app/admin/setup/page.tsx`)
- 연결 테스트 성공 시 시트 이름을 `custom_teacher_name` 쿠키에 저장합니다.
- 설정 초기화(`handleReset`) 시 해당 쿠키도 함께 삭제하도록 로직을 강화했습니다.

### C. 관리자 대시보드 가시성 확보 (`app/admin/page.tsx`)
- 관리자 홈 상단에 현재 어떤 선생님의 시트가 연결되어 있는지 보여주는 **상태 표시 바(Status Bar)**를 추가했습니다.
- 여기서 즉시 **[기본 시스템으로 복구하기]** 기능을 실행할 수 있습니다.

### D. 전역 상태 알림 및 복구 기능 (`components/BackendStatus.tsx`)
- 사이트 어디서든(랜딩 페이지 등) 하단에 현재 개인 연구소 모드임을 알리는 플로팅 바를 추가했습니다.
- 매직 링크를 타고 들어온 사용자가 자신이 누구의 시트를 쓰고 있는지 확인하고, 필요시 초기화할 수 있습니다.

### E. 매직 링크 자동 인식 강화 (`components/LayoutClientWrapper.tsx`)
- 매직 링크로 접속 시 단순히 URL만 저장하는 것이 아니라, 백엔드에 즉시 핑(Ping)을 보내 선생님 성함을 가져와 저장하도록 프로세스를 개선했습니다.

## 3. 해결된 문제
- "지금 이 시트가 내 것인가? 아니면 기본 시트인가?"에 대한 혼란 원천 차단.
- 매직 링크 사용 후 다시 원래 기본 환경으로 돌아가고 싶을 때 쿠키를 수동으로 지워야 했던 불편함 해소.

## 4. 향후 관리 사항
- `custom_teacher_name` 쿠키가 없는 예전 설정 사용자의 경우, 다시 한번 '연결 테스트'를 수행하면 이름이 정상 등록됩니다.
