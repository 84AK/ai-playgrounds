# 작업 로그: 과제 제출 UI 전면 개편 (모달 -> 인페이지 섹션) (2026-03-13)

## 1. 개편 배경
- 기존 팝업(모달) 방식이 화면 크기에 따라 내용이 잘리거나, 가이드와 업로드 화면을 동시에 보기 어렵다는 피드백 반영.
- 학습 가이드 확인과 과제 제출을 단일 흐름으로 통합하여 가장 효과적인 UX 제공 목적.

## 2. 주요 작업 내용
- **UploadHomework.tsx (컴포넌트)**:
    - `fixed`, `absolute` 모달 스타일을 모두 제거하고 일반 `section`으로 재구성.
    - 2컬럼 레이아웃 적용: 왼쪽은 업로드 영역, 오른쪽은 업로드 가이드 카드로 분리하여 정보 전달력 강화.
    - `id="submission-section"` 부여 및 `scroll-mt-32` 설정을 통해 부드러운 스크롤 타겟 지정.
- **CourseSubmissionTrigger.tsx (트리거)**:
    - 클릭 시 모달 오픈 대신 `scrollIntoView` API를 사용하여 하단 제출 섹션으로 자동 이동하도록 기능 변경.
- **page.tsx (페이지)**:
    - 학습 가이드 하단에 `UploadHomework` 섹션을 고정적으로 노출하도록 통합.

## 3. 기대 효과
- **가시성 확보**: 가이드와 업로드 창이 한눈에 들어와 사용자가 혼동 없이 작업 가능.
- **안정성**: 팝업창 잘림 현상 등 브라우저 뷰포트 관련 UI 이슈 완전 해결.
- **편의성**: 학습을 마친 후 자연스럽게 아래로 내려가거나, 위에서 제출 버튼 하나로 즉시 이동 가능.
