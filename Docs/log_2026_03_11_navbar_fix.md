# 작업 로그: 네비게이션 메뉴 레이아웃 깨짐 현상 해결 (2026-03-11)

## 1. 문제 증상 (Issue)
- **현상**: `ai-playgrounds` 앱의 상단 네비게이션 메뉴가 768px(md) ~ 1024px(lg) 사이의 해상도에서 메뉴 항목들의 텍스트가 줄바꿈되거나, 로고/프로필 영역과 겹쳐서 "찌그러져" 보이는 현상 발생.
- **원인**: 
  - 메뉴 항목이 6개로 많아 가로 너비를 많이 차지함.
  - 기존 브레이크포인트(`md`: 768px) 기준에서는 이 항목들을 한 줄에 담기에 공간이 부족하여 텍스트 래핑이 발생함.
  - 로고 및 프로필 영역에 압축 방지(`shrink-0`) 설정이 없어 메뉴 공간 확보를 위해 함께 압축됨.

## 2. 해결 방법 (Solution)
- **[건축가/작업자]** `components/Navbar.tsx` 수정:
  - **텍스트 줄바꿈 방지**: 각 메뉴 링크에 `whitespace-nowrap` 클래스 추가.
  - **브레이크포인트 상향 조정**: 데스크톱 가로 메뉴 표시 기준을 `md`에서 `lg`(1024px)로 변경하여, 공간이 충분한 환경에서만 가로 메뉴를 노출.
  - **모바일 메뉴 전환**: 1024px 미만에서는 깔끔한 'MENU' 버튼(햄버거 스타일)으로 전환되도록 설정 변경 (`lg:hidden`).
  - **영역 보호**: 로고(`Link`)와 사용자 프로필 섹션에 `shrink-0`을 적용하여 메뉴 항목이 많더라도 핵심 UI가 찌그러지지 않도록 보호.

## 3. 구현 세부 사항 (Implementation Details)
```tsx
// 주요 변경 부분
<Link href="/" className="... shrink-0">...</Link> // 로고 보호
<ul className="hidden lg:flex ..."> // 데스크톱 메뉴 노출 기준 상향
  <Link className="... whitespace-nowrap">...</Link> // 텍스트 줄바꿈 방지
</ul>
<div className="... shrink-0">...</div> // 프로필 영역 보호
<button className="lg:hidden ...">...</button> // 모바일 버튼 노출 기준 상향
```

## 4. 결과 및 확인 (Verification)
- **1024px 이상**: 모든 메뉴가 한 줄로 쾌적하게 배치되며 텍스트가 깨지지 않음.
- **1024px 미만**: 복잡한 가로 메뉴 대신 세련된 모바일 메뉴 버튼이 노출되어 UI 일관성 유지.
- **공통**: 화면이 좁아지더라도 로고와 프로필이 찌그러지지 않고 형태를 유지함.

## 5. 향후 고려 사항
- 메뉴 항목이 더 추가될 경우, 1280px(xl) 브레이크포인트를 활용하거나 메뉴 폰트 크기를 미세하게 조정하는 방안 검토 필요.
