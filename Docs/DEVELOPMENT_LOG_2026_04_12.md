# 🖋️ 프로젝트 작업 로그 (2026-04-12)

## 📋 개요
오늘의 핵심 목표는 **"교사별 독립 데이터 환경(Multi-tenant)"** 구축과 **"스마트 과제 관리 시스템"** 완성이었습니다. 이를 통해 각 교사가 자신의 구글 시트와 드라이브를 사용하여 독립적으로 인공지능 연구소를 운영할 수 있게 되었습니다.

---

## 🛠️ 주요 구현 및 수정 사항

### 1. 멀티 테넌트 분산화 아키텍처 (Decentralization)
- **쿠키 기반 동적 라우팅**: `custom_gs_url` 및 `custom_admin_password` 쿠키를 사용하여 사용자의 브라우저 상태에 따라 백엔드(Apps Script) 주소를 동적으로 결정하도록 구현.
- **API 전면 개편**: `/api/course/content`, `/api/course/save` 등 모든 서버 사이드 엔드포인트가 쿠키를 참조하여 개개인의 전용 백엔드로 요청을 보내도록 수정.
- **매직 링크(Magic Link) 시스템**: 학생이 별도 설정 없이 선생님의 시트와 연결될 수 있도록 URL 파라미터를 통한 자동 온보딩 기능 구현.

### 2. 범용 커리큘럼 뷰어 (Universal Viewer)
- **동적 트랙 지원**: MBTI, POSE 등 고정된 트랙 외에도 교사가 추가한 모든 새로운 트랙(예: AI)을 자동으로 인식하고 보여주는 `app/curriculum/[track]/[weekId]` 페이지 신설.
- **제출 시스템 통합**: 어떤 트랙에서도 과제를 제출할 수 있는 범용 `UploadHomework` 및 `CourseSubmissionTrigger` 컴포넌트 개발 및 이식.

### 3. 스마트 과제 관리 시스템 (GAS V7.8)
- **계층형 자동 폴더링**: `[트랙] -> [주차] -> [학년반]` 순으로 구글 드라이브 내 폴더를 자동 생성하고 분류하는 로직 구현.
- **지능형 재제출(수정) 로직**: 학생이 과제를 다시 올릴 경우, 기존 파일을 자동으로 검색하여 **휴지통(Trash)**으로 보내고 최신 파일로 교체하는 로직 구현.
- **UI 피드백**: 제출 완료 시 버튼 이름을 `과제 수정하기`로 변경하고 현재 제출된 파일명을 표시하여 사용자 경험 개선.

### 4. 관리자 셋업 고도화
- **템플릿 엔진 업데이트**: `lib/gasTemplate.ts`를 버전 7.8로 업그레이드하여, 모든 신규 관리자가 최신 기능을 즉시 복사하여 사용할 수 있게 함.

## 🔒 보안 및 기능 검토 결과 (Pre-Commit Audit)

### 1. 보안성 (Security)
- **API Key 노출 없음**: 모든 민감한 정보(Apps Script URL, Folder ID 등)는 환경 변수와 쿠키를 통해 관리되며, 클라이언트 코드에 하드코딩되지 않음.
- **인증 로직**: 선생님별 개인 비밀번호(`custom_admin_password`)를 통한 2차 인증 체계 확인.
- **보안 규칙 준수**: `vibecodingsecurityrules.md`를 기반으로 사용자 입력값 검증 및 안전한 데이터 처리 확인.

### 2. 기능성 (Functional)
- **상호 운용성**: 마스터 비밀번호와 커스텀 비밀번호가 충돌 없이 공존하며 작동함.
- **예외 처리**: Apps Script URL 실패 시나 시트 데이터 누락 시에 대한 에러 핸들링 보강.
