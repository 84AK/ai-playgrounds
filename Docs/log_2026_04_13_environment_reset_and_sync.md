# 작업 로그: 시스템 환경 초기화 및 매직 링크 풀-싱크 구현 (2026-04-13)

## 1. 개요
사용자가 테스트 중인 개별 설정에서 원본 시스템(Vercel 설정)으로 언제든지 돌아갈 수 있도록 '초기화' 기능을 강화하고, 관리자 매직 링크를 통한 동료 교사 초대 시 모든 백엔드 환경이 한 번에 동기화되도록 개선했습니다.

## 2. 주요 변경 사항

### A. 시스템 초기화 API 및 버튼 추가 (Environment Reset)
- **API**: `/api/admin/reset-settings` (POST)를 신설하여 브라우저에 저장된 모든 커스텀 쿠키(`custom_gs_url`, `custom_teacher_name`, `admin_session` 등)를 한 번에 삭제하도록 구현했습니다.
- **관리자 화면**: 대시보드 상단에 **"현재 개인화된 설정을 사용 중입니다. [원본 시스템으로 복구하기]"** 배너를 추가하여 직관적인 복구 경로를 제공합니다.
- **로그인 화면**: 학생용 로그인 모달 하단에도 **"🔄 System Reset"** 링크를 추가하여, 선생님이 테스트 후 설정을 안 지웠을 경우 학생들이 직접 원본 시스템으로 돌아갈 수 있게 조치했습니다.

### B. 관리자 매직 링크 '완전체' 동기화 (Full Sync Magic Link)
- **문제**: 이전 링크는 이름과 세션만 넘겨주어, 동료 교사가 접속하면 엉뚱한(데모) 시트를 보게 되는 현상이 있었습니다.
- **해결**: 매직 링크 생성 시 현재의 `gsUrl`, `folderId`, `notionDbId`, `notionPriority` 등 모든 핵심 설정을 URL 파라ми터에 포함하도록 수정했습니다.
- **로그인 처리**: `/api/admin/magic-login` 핸들러가 이 모든 정보를 수신하여 클릭 한 번으로 동료 교사의 브라우저를 완벽하게 동기화합니다.

### C. UX 개선
- 관리자 및 설정 페이지에서 현재 상태(원본 vs 커스텀)를 인지할 수 있는 상태 표시등을 강화했습니다.

## 3. 해결된 문제
- "로그인이 안 된다(이름 없음)" 에러의 근본 원인인 **'백엔드 환경 불일치'** 문제를 매직 링크 풀-싱크와 초기화 버튼으로 완벽하게 해결했습니다.
