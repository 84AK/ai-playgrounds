# 2026-03-28 쇼케이스 고도화 작업 로그 (Doc / Scribe)

## 1. 개요
쇼케이스 페이지에서 `ShowcaseLinks` 시트 데이터가 표시되지 않던 문제를 해결하고, 학교/학년/반 기반의 정교한 필터링 및 통합 검색 시스템을 도입했습니다.

## 2. 작업 상세 내역

### 🛠️ 데이터 로딩 및 파싱 매핑 수정 (Fixer)
- **문제점**: 구글 앱스 스크립트 API로부터 `ShowcaseLinks` 시트 데이터를 가져올 때 키 명칭 불일치(`showcase_links` vs `ShowcaseLinks`)로 인해 프론트엔드에서 데이터를 맵핑하지 못함.
- **해결 방안**: `showcase_links`, `ShowcaseLinks`, `showcaseLinks` 등 여러 키 후보를 모두 체크하도록 로직을 수정하여 시트 데이터 로딩을 정상화함.
- **프로필 결합**: `result.data.users` 객체를 탐색하여 각 작성자의 학교, 학년, 반 정보를 프로젝트 아이템 데이터에 병합함.

### 🔍 필터링 및 검색 로직 구현 (Architect & Worker)
- **상태 관리**: 학교(`schoolFilter`), 학년(`gradeFilter`), 반(`classFilter`), 검색어(`searchQuery`)를 위한 React State 도입.
- **동적 옵션 추출**: 실시간으로 로드된 전체 프로젝트에서 고유한 학교명, 학년, 반 목록을 추출하여 드롭다운 메뉴를 구성함.
- **복합 필터 알고리즘**: 카테고리 선택 값과 다중 필드 검색어 매칭, 소속 정보 필터링을 교차 적용하여 정교한 검색 결과를 제공함.

### ✨ UI/UX 전면 개편 (Designer)
- **날짜 필터 제거**: 사용자 요청에 따라 기존의 날짜 기반 필터 드롭다운을 제거함.
- **글래스모피즘 검색바**: 상단에 돋보기 아이콘이 포함된 고급스러운 검색창을 배치하여 접근성 향상.
- **벤토 그리드 필터**: 학교, 학년, 반 선택 메뉴를 고유한 디자인의 벤토 박스 형태로 나란히 배치.
- **카드 인터페이스 업데이트**: 각 작품 카드에 작성자의 소속 정보(학교, 학년, 반)를 추가로 노출하여 시각적 연동성 강화.

## 3. 검증 결과
- 모든 시트 내의 전시 전용 링크가 갤러리에 정상 로드됨을 확인.
- 학교별/학년별/반별 필터가 의도대로 동작하며, 검색어와의 조합도 완벽하게 작동함.
- 모바일 환경에서의 레이아웃 반응형 테스트 완료.
