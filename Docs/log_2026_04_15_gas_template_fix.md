# 작업 로그 (2026-04-15) - 쇼케이스 컬럼 밀림 현상 해결

## 1. 개요
쇼케이스 페이지에서 새 프로젝트 등록 시 스프레드시트의 데이터가 한 칸씩 밀려 저장되는 현상을 발견하고 이를 수정함.

## 2. 문제 현상
- 프로젝트 등록 시 데이터가 H열(visible)이 아닌 I열에 저장됨.
- 이로 인해 `getAllMbtiData` 액션에서 데이터를 불러올 때 'visible' 상태값을 제대로 읽지 못해 목록에 표시되지 않음.
- 등록 직후 새로고침 시 세션 검증과 맞물려 로그인 페이지로 리다이렉트되는 현상이 동반될 가능성이 있음.

## 3. 원인 분석
- `lib/gasTemplate.ts` 내의 GAS 템플릿 코드 오류.
- `registerShowcaseLink` 액션의 `appendRow` 로직에서 실수로 `'Empty'` 컬럼(8번째)이 추가되어 있었음.

## 4. 해결 내용
- **대상 파일**: 
    1. `lib/gasTemplate.ts` (일반 관리자용 템플릿)
    2. 사용자 직접 제공 코드 (슈퍼 관리자용 V9.8.1)
- **수정 상세**:
    - `registerShowcaseLink` 액션의 헤더 행에서 `'Empty'` 항목 제거.
    - `appendRow` 데이터 배열의 8번째 인덱스에 있던 빈 문자열(`''` 또는 `""`) 제거.
    - 전체 구조를 8개 컬럼으로 통일: `['Timestamp', 'Author', 'Title', 'Description', 'Url', 'Password', 'category', 'visible']`
    - (추가 수정) 슈퍼 관리자용 코드에서 누락될 수 있는 컬럼 인덱스 정합성 재확인.

## 5. 결과 및 기대 효과
- 새로운 프로젝트 등록 시 'visible' 값이 H열에 정확히 저장되어 필터링 로직(`getAllMbtiData`)이 정상 작동함.
- 쇼케이스 목록에 새 프로젝트가 누락 없이 즉시 표시됨.
- 슈퍼 관리자용 최신 버전(V9.8.1)의 전체 코드 무결성 확보.

## 6. 향후 주의사항
- GAS 코드를 직접 수정하거나 템플릿을 변경할 때는 스프레드시트의 실제 헤더 순서와 배열의 인덱스가 1:1로 일치하는지 반드시 대조 확인 필요.
- 관리자 화면에서 수정된 코드를 복사하여 배포한 후, 기존 I열에 잘못 저장된 데이터는 수동으로 H열로 이동 처리가 필요함.
- 6번 '쇼케이스 링크 관리' 블록의 `appendRow` 배열 길이를 항상 8개로 유지해야 함.
