# 작업 로그 (Work Log) - 2026.04.11

## 📋 개요
기존의 하드코딩된 4주차 중심의 교육과정 시스템을 데이터베이스(Google Sheets) 중심의 동적 교육과정 시스템으로 전면 개편하였습니다. 이제 관리자는 코스 트랙과 주차를 자유롭게 추가하고 편집할 수 있으며, 학생 대시보드는 이를 실시간으로 반영합니다.

## 🛠 구현 내용

### 1. 백엔드 (Google Apps Script) 고도화
- **교육과정 구조화**: `CourseContents` 시트를 단일 소스로 통합하고, `Title` 컬럼을 추가하여 동적 렌더링 지원.
- **주차 번호 체계 통합**: 트랙별 주차(mbti_week1 등)가 아닌 절대 주차(1~8주차 이상) 체계로 통합.
- **과제 보관 구조 표준화**: 구글 드라이브 폴더명을 `${week}주차` 형식으로 통일하여 연속성 확보.
- **진도 데이터 동기화**: `Progress` 시트의 컬럼을 `week1`~`week8`로 통합 관리하도록 로직 수정.

### 2. 프론트엔드 공유 라이브러리 (`lib/courseContent.ts`)
- `getCourseStructure()`: 전체 트랙과 주차 정보를 동적으로 가져오는 기능 추가.
- `getCourseContent()`, `saveCourseContent()`: 새로운 `Title` 필드 및 동적 트랙 파라미터 대응.

### 3. API 라우트 고도화
- `GET /api/course/structure`: 현재 교육과정의 모든 트랙/주차 리스트 반환.
- `GET /api/course/content` & `POST /api/course/save`: 4주차 제한 해제 및 주차 정보 동적 처리.

### 4. 관리자 대시보드 (`app/admin/course`)
- **동적 네비게이션**: 하드코딩된 트랙 리스트를 제거하고 API를 통해 실시간 구조 로드.
- **주차 등록 기능**: 관리자 UI에서 새로운 트랙이나 주차를 즉시 등록할 수 있는 기능 구현.
- **에디터 개선**: 콘텐츠 편집 시 `Title` 필드를 함께 수정할 수 있도록 UI 업데이트.

### 5. 학생 대시보드 (`StudyLabPanel`)
- **동적 탭 시스템**: 트랙 정보에 따라 상단 탭이 자동으로 생성되도록 리팩토링.
- **진도율 계산 로직**: 통합된 `week{i}` 키를 기반으로 전체 주차 진도율을 정확히 계산.
- **UI 최적화**: 트랙 이름과 주차 제목이 교육과정 설정에 따라 유연하게 표시됨.

### 6. 과제 제출 시스템
- **통합 폴더링**: MBTI, Pose 트랙 구분 없이 절대 주차 숫자에 맞춰 구글 드라이브 폴더 생성.
- **진도 확인 로직**: 제출 후 진도 업데이트 확인 시 통합 주차 키(`week1`~`week8`) 사용.

### 7. 쇼케이스 (`Showcase`)
- **동적 과제 선택**: 전시 등록 시 불러올 수 있는 과제 목록을 교육과정 구조에 따라 동적으로 생성.

## ⚠️ 해결된 문제 (Hotfixes)
- **문제**: Pose 트랙이 MBTI 이후 5-8주차로 이어질 때 진도율이 0%로 표시되는 현상.
- **해결**: 트랙별 하드코딩된 키 대신 `courseStructure`에서 추출한 주차 정보를 기반으로 진도 키를 매핑하도록 수정.
- **문제**: 과제 제출 시 트랙 이름이 폴더명에 포함되어 연속성이 깨지는 문제.
- **해결**: GAS 로직에서 트랙 정보를 제외하고 `${week}주차` 형식으로 폴더명을 고정.

## 🚀 향후 과제
- 기존 `Progress` 시트의 데이터(mbti_week1 등)를 새로운 `week1` 형식으로 옮기는 일회성 마이그레이션 필요.
- 관리자 화면에서 주차 삭제 기능 추가 검토.
