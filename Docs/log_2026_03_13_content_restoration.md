# 작업 로그: 학습 차시 내용 복구 및 서버 컴포넌트 환원 (2026-03-13)

## 1. 발생한 문제
- 학습 차시 페이지(`app/course/[weekId]/page.tsx`)를 클라이언트 컴포넌트로 변경한 후, 학습 내용이 "진행 중인 문서가 없습니다"라고 표시됨.
- 원인: 기존 서버 로직에 포함되어 있던 '로컬 파일 폴백(Fallback)' 기능(스프레드시트에 데이터가 없을 경우 `Docs/` 폴더의 파일을 읽어오는 기능)이 브라우저 환경에서는 파일 시스템 접근 권한 문제로 작동하지 않음.

## 2. 해결 방법
- **페이지 컴포넌트 서버화**: `page.tsx`에 다시 `async` 서버 컴포넌트 구조를 적용하여 서버 단에서 로컬 파일과 스프레드시트를 모두 조회할 수 있도록 복구.
- **인터랙션 분리**: 모달 팝업 상태 관리(`useState`)가 필요한 부분만 별도의 클라이언트 컴포넌트(`CourseSubmissionTrigger.tsx`)로 분리하여 구현.
- **데이터 조회 우선순위**: `lib/courseContent.ts`의 로직을 그대로 활용하여, 스프레드시트에 데이터가 없더라도 로컬 마크다운 파일이 있으면 정상적으로 출력되도록 보장.

## 3. 변경 사항
- `[MODIFY] app/course/[weekId]/page.tsx`: 서버 컴포넌트로 환원 및 데이터 로딩 로직 복구.
- `[NEW] app/course/[weekId]/CourseSubmissionTrigger.tsx`: 제출 버튼 및 모달 트리거 전용 클라이언트 컴포넌트 추가.

## 4. 향후 참고 사항
- 학습 내용은 `Docs/mbti_week{N}.md` 파일에 저장되어 있으며, 스프레드시트의 `course_contents` 시트에서 A열(track)과 B열(week)을 매칭하여 관리할 수도 있습니다.
- 현재는 스프레드시트가 비어 있어도 로컬 파일 덕분에 정상 출력됩니다.
