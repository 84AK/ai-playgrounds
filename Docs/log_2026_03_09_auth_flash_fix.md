# 작업 로그 - 로그인 UI 플래시(깜빡임) 현상 수정

## 일시
2026년 3월 9일

## 작업 역할
Couselor(상담가), Fix(해결사), Doc(서기)

## 문제 현상 (Issue)
- 페이지를 이동하거나 새로고침할 때, 이미 로그인한 상태임에도 불구하고 로그인 창이 약 0.1~0.2초간 나타났다 사라지는 현상 발생.
- 사용자 경험을 저해하고 시스템이 불안정하게 느껴지는 원인이 됨.

## 원인 분석 (Root Cause)
- **리액트 하이드레이션(Hydration) 지연**: `useLocalProfile` 훅이 `useEffect`를 통해 상태를 동기화하기 전까지 `null` 값을 반환함.
- **인증 가드 로직**: `GlobalAuthGuard`가 상태 업데이트를 기다리지 않고 `null`인 상태를 확인하자마자 즉시 로그인 폼(`setStep("login")`)을 렌더링하여 발생한 레이스 컨디션.

## 해결 방법 (Solution)

### 1. 동기식 로컬 프로필 체크 (`GlobalAuthGuard.tsx`)
- `useEffect`가 실행되는 즉시 `readLocalProfile()` 함수를 **동기적**으로 호출하여 브라우저의 `localStorage`를 직접 확인합니다.
- 상태(state)가 바뀌기 전이라도 로컬 스토리지에 유효한 정보가 있다면 로그인 창을 띄우지 않도록 우선순위를 조정했습니다.

### 2. 하이드레이션 보호 강화
- `isMounting` 상태와 `step` 전환 로직을 최적화하여, 유저 정보가 확실히 없다고 판단될 때만 로그인 화면이 보이도록 개선했습니다.

## 결과 및 검증
- `npm run build`를 통해 코드 무결성 확인 완료.
- 페이지 전환 및 새로고침 시 로그인 창 노출 없이 부드럽게 페이지가 로딩됨을 확인했습니다.
- 로그아웃 후에는 정상적으로 로그인 창이 나타나는 기본 기능도 유지됨을 확인했습니다.
