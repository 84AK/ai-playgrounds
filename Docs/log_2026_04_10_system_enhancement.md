## 🛠️ 수정 및 해결 내용 (Fix & Solve)

### 1. 과제 중복 제출 방지 및 수정 시스템 (Option B)
- **문제**: 학생이 동일 주차에 파일을 여러 번 제출할 경우 구글 드라이브에 중복 파일이 쌓여 관리가 어려움.
- **해결**: 
    - `updated_gas_code.js` (V5.7): 과제 업로드 전 해당 주차 폴더 전체를 검색하여 해당 학생의 이전 파일을 찾아 휴지통으로 이동(`setTrashed`)시키는 로직 추가.
    - 학년/반 정보를 변경하여 제출하더라도 닉네임을 기반으로 이전 파일을 추적하도록 고도화.
- **UI**: 제출 완료 페이지에서 버튼이 "과제 수정하기"로 동적 변경되며, 덮어쓰기 전 사용자에게 확인(Confirm) 창 제공.

### 2. 쇼케이스 라이브 프리뷰 이미지 로딩 버그
- **문제**: MBTI 결과 화면 등 자바스크립트에 의해 동적으로 로드되는 이미지가 프리뷰 창에서 '엑박'으로 나타남.
- **원인**: 기존 엔진이 HTML 내 정적 태그만 치환하고 JS/CSS 소스 내부의 에셋 경로는 치환하지 못했음.
- **해결**:
    - `lib/zipParser.ts`: 모든 소스 코드(HTML, JS, CSS)를 전수 조사하여 이미지 경로 패턴을 브라우저 가상 Blob URL로 자동 치환하는 `applyAssetUrls` 로직 적용.
